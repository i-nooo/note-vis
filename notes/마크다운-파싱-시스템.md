이 프로젝트의 마크다운 파싱 시스템은 **빌드 타임**과 **런타임** 두 단계로 구성된다.

## 1. 빌드 타임 파싱

`scripts/parse-notes.js`가 `notes/` 폴더의 마크다운 파일들을 파싱하여 note들 간 관계 그래프 데이터를 생성한다.

**실행 시점**

- `npm run build` 실행 시 자동으로 실행
- `npm run parse-notes`로 수동 실행

**파싱 과정**

```
notes/*.md → parse-notes.js → src/data/notes.json
```

### Frontmatter 파싱

각 마크다운 파일 상단의 YAML frontmatter에서 메타데이터를 추출한다.

| 필드    | 정규식                                                    | 설명      |
| ------- | --------------------------------------------------------- | --------- |
| title   | `title:\s*["']?([^"'\n]+)["']?`                           | 노트 제목 |
| tags    | `tags:\s*\[(.*?)\]`                                       | 태그 배열 |
| related | `related:\s*\[(.*?)\]`                                    | 관련 노트 |
| created | `created(?:\s+date)?:\s*["']?([^"'\n]+)["']?`             | 생성일    |
| updated | `(?:updated\|edited)(?:\s+date)?:\s*["']?([^"'\n]+)["']?` | 수정일    |

### Wiki Link 추출

본문에서 `[[링크]]` 패턴을 찾아 노트 간 연결 관계를 추출한다.

```javascript
const REGEX = {
  wikiLink: /\[\[([^\]]+)\]\]/g,
};
```

파이프 링크 `[[target|display]]`도 지원하며, target 부분만 링크 대상으로 사용한다.

### 출력 데이터 구조

```json
{
  "nodes": [
    {
      "id": "파일명",
      "title": "노트 제목",
      "tags": ["태그1", "태그2"],
      "content": "frontmatter 제외한 본문",
      "dateCreated": "2025-01-01",
      "dateUpdated": "2025-01-15"
    },
    {
      "id": "tag:태그명",
      "title": "#태그명"
    }
  ],
  "links": [
    { "source": "노트A", "target": "노트B", "type": "mention" },
    { "source": "노트A", "target": "tag:태그", "type": "tag" }
  ]
}
```

### 깨진 링크 처리

존재하지 않는 노트를 참조하는 링크는 `broken: true` 플래그가 추가된다.

## 2. 런타임 렌더링

`src/utils/markdown/` 모듈이 노트 상세 페이지에서 마크다운을 HTML로 렌더링한다. 사용자가 노트를 클릭하면 저장된 마크다운 본문(`content`)이 이 모듈을 통해 실시간으로 HTML로 변환된다.

### 모듈 구조

```
src/utils/markdown/
├── index.ts          # renderMarkdown() 메인 함수
├── extensions/
│   ├── index.ts      # extension export
│   ├── wikilink.ts   # [[wiki link]] 커스텀 토크나이저/렌더러
│   └── callout.ts    # > [!NOTE] 콜아웃 처리
├── renderer.ts       # 코드 하이라이팅, 링크 정책
├── helpers.ts        # 유틸리티 함수
├── constants.ts      # 설정값
└── types.ts          # TypeScript 타입 정의
```

### 메인 함수: renderMarkdown()

`index.ts`의 `renderMarkdown()` 함수가 전체 렌더링 파이프라인을 관리한다.

```typescript
export function renderMarkdown(
  markdown: string,
  brokenLinks?: Set<string>,
): RenderResult {
  // 1. 깨진 링크 컨텍스트 설정
  setBrokenLinks(brokenLinks || new Set());

  // 2. 각주 추출 및 처리
  const footnotes = extractFootnotes(markdown);
  let contentWithoutFootnoteDefs = removeFootnoteDefinitions(markdown);
  if (footnotes.length > 0) {
    contentWithoutFootnoteDefs = replaceFootnoteReferences(
      contentWithoutFootnoteDefs,
    );
  }

  // 3. marked로 HTML 변환
  const html = marked.parse(contentWithoutFootnoteDefs) as string;
  const footnotesHtml = renderFootnotesHtml(footnotes);

  // 4. 프로덕션에서 DOMPurify 적용
  if (import.meta.env.DEV) {
    return { content: html, footnotes: footnotesHtml };
  }
  return {
    content: DOMPurify.sanitize(html, SANITIZE_OPTIONS),
    footnotes: DOMPurify.sanitize(footnotesHtml, SANITIZE_OPTIONS),
  };
}
```

### marked 라이브러리 확장

[marked](https://marked.js.org/)는 마크다운을 HTML로 변환하는 라이브러리다. 이 프로젝트에서는 커스텀 extension을 통해 Obsidian 스타일 문법을 지원한다.

**초기화 (setupMarked)**:

```typescript
function setupMarked(): void {
  const renderer = createRendererWithLinkPolicy();
  marked.use({ renderer });
  marked.use({
    extensions: [
      wikiLinkTokenizer, // [[link]] 토큰화
      wikiLinkRenderer, // [[link]] HTML 변환
      calloutTokenizer, // > [!NOTE] 토큰화
      calloutRenderer, // > [!NOTE] HTML 변환
    ],
  });
  marked.setOptions({
    gfm: true, // GitHub Flavored Markdown
    breaks: true, // 줄바꿈을 <br>로 변환
  });
}

setupMarked(); // 모듈 로드 시 한 번 실행
```

### Wiki Link Extension 상세

Obsidian의 `[[wiki link]]` 문법을 지원한다.

#### Tokenizer (토큰화)

마크다운 파싱 단계에서 `[[...]]` 패턴을 찾아 커스텀 토큰으로 변환한다.

```typescript
export const wikiLinkTokenizer: TokenizerExtension = {
  name: "wikilink",
  level: "inline", // 인라인 요소로 처리

  // 패턴 시작 위치 탐색 (성능 최적화)
  start(src: string) {
    const i = src.indexOf("[[");
    return i === -1 ? undefined : i;
  },

  // 실제 토큰화
  tokenizer(src): WikiLinkToken | undefined {
    // [[target]] 또는 [[target|별칭]] 패턴 매칭
    const match = /^\[\[([^\]|]+?)(?:\|([^\]]+?))?\]\]/.exec(src);
    if (!match) return undefined;

    const [, rawName, alias] = match;
    const text = alias ? alias.trim() : rawName.trim();
    const href = `/node/${slugifyNode(rawName)}`;

    return {
      type: "wikilink",
      raw: match[0], // 원본 문자열 (marked가 소비)
      href,
      text,
    };
  },
};
```

**예시**:

- `[[React.memo]]` → `{ href: '/node/React.memo', text: 'React.memo' }`
- `[[React.memo|메모]]` → `{ href: '/node/React.memo', text: '메모' }`

#### Renderer (HTML 변환)

토큰을 HTML `<a>` 태그로 변환한다.

```typescript
export const wikiLinkRenderer: RendererExtension = {
  name: "wikilink",
  renderer(token: Tokens.Generic) {
    const wikiToken = token as WikiLinkToken;
    const brokenLinks = getBrokenLinks();

    // URL에서 노트 ID 추출
    const targetId = wikiToken.href.replace("/node/", "").replace(/%20/g, " ");
    const isBroken = brokenLinks.has(decodeURIComponent(targetId));

    // 깨진 링크는 class 추가 + href를 #으로
    const className = isBroken ? ' class="broken-link"' : "";
    const href = isBroken ? "#" : wikiToken.href;

    return `<a href="${href}"${className}>${escapeHtml(wikiToken.text)}</a>`;
  },
};
```

**깨진 링크 처리**:

- 존재하지 않는 노트를 참조하면 `class="broken-link"` 추가
- CSS로 취소선, 빨간색 등 스타일링 가능

### Callout Extension 상세

Obsidian 스타일의 콜아웃(인용 블록 확장) 문법을 지원한다.

#### 문법

```markdown
> [!NOTE] 커스텀 제목
> 콜아웃 내용
> 여러 줄 가능
```

#### Tokenizer

```typescript
export const calloutTokenizer: TokenizerExtension = {
  name: "callout",
  level: "block", // 블록 요소로 처리

  start(src: string) {
    const i = src.indexOf("> [!");
    return i === -1 ? undefined : i;
  },

  tokenizer(src): CalloutToken | undefined {
    // 인용 블록 전체 매칭
    const cap = /^(?:> ?(.*)\n?)+/.exec(src);
    if (!cap) return undefined;

    // 각 줄에서 '> ' 제거
    const lines = cap[0]
      .trimEnd()
      .split("\n")
      .map((l) => l.replace(/^>\s?/, ""));

    // 첫 줄에서 [!TYPE] 패턴 확인
    const head = lines[0];
    const headMatch = /^\[!(NOTE|INFO)\]([+-])?(?:\s+(.*))?$/i.exec(head);
    if (!headMatch) return undefined;

    const [, typeRaw, collapseFlag, customTitle] = headMatch;
    const calloutType = typeRaw.toUpperCase() as CalloutKind;

    // 접기 상태: '-'면 접힘, '+'면 펼침, 없으면 undefined
    const collapsed =
      collapseFlag === "-" ? true : collapseFlag === "+" ? false : undefined;

    // 본문 (첫 줄 제외)을 재귀적으로 파싱
    const body = lines.slice(1).join("\n");
    const bodyTokens = this.lexer.blockTokens(body, []);

    return {
      type: "callout",
      raw: cap[0],
      calloutType,
      title: customTitle?.trim(),
      collapsed,
      tokens: bodyTokens,
    };
  },
};
```

#### Renderer

```typescript
export const calloutRenderer: RendererExtension = {
  name: "callout",
  renderer(token: Tokens.Generic) {
    const calloutToken = token as CalloutToken;
    const cfg = CALLOUTS[calloutToken.calloutType]; // 설정 조회
    const title = calloutToken.title || cfg.defaultTitle;

    // 접기 상태에 따라 open 속성 결정
    const detailsOpen =
      calloutToken.collapsed === undefined
        ? " open"
        : calloutToken.collapsed
          ? ""
          : " open";

    // 본문 토큰을 재귀적으로 렌더링
    const inner = this.parser.parse(calloutToken.tokens);

    return `
<aside class="callout ${cfg.className}" role="note" aria-label="${cfg.ariaLabel}">
  <details class="callout-details"${detailsOpen}>
    <summary class="callout-header">
      <span class="callout-title">${title}</span>
    </summary>
    <div class="callout-content">
${inner}
    </div>
  </details>
</aside>`.trim();
  },
};
```

**Callout 설정** (`constants.ts`):

```typescript
export const CALLOUTS: Record<CalloutKind, CalloutConfig> = {
  NOTE: {
    className: "callout-note",
    defaultTitle: "노트",
    ariaLabel: "Note",
  },
  INFO: {
    className: "callout-info",
    defaultTitle: "정보",
    ariaLabel: "Info",
  },
};
```

### 각주(Footnotes) 처리 상세

마크다운 각주를 클릭 가능한 링크로 변환한다.

#### 처리 단계

**1. 각주 정의 추출**:

```typescript
function extractFootnotes(markdown: string): Array<Footnote> {
  const footnotes: Array<Footnote> = [];
  // 각주 정의 패턴: [^label]: content
  const footnoteMatches = markdown.match(/^\[\^([^\]]+)\]:\s*(.*)$/gm);

  if (footnoteMatches) {
    footnoteMatches.forEach((matchStr) => {
      const parts = matchStr.match(/^\[\^([^\]]+)\]:\s*(.*)$/);
      if (parts) {
        footnotes.push({
          label: parts[1].trim(),
          content: parts[2].trim(),
        });
      }
    });
  }
  return footnotes;
}
```

**2. 본문에서 각주 정의 제거**:

```typescript
function removeFootnoteDefinitions(markdown: string): string {
  return markdown.replace(/^\[\^([^\]]+)\]:\s*.*$/gm, "").trim();
}
```

**3. 참조를 링크로 변환**:

```typescript
function replaceFootnoteReferences(content: string): string {
  return content.replace(
    /\[\^([^\]]+)\]/g,
    '<sup class="footnote-ref"><a href="#footnote-$1" id="footnote-ref-$1" class="footnote-link">$1</a></sup>',
  );
}
```

**4. 각주 목록 HTML 생성**:

```typescript
function renderFootnotesHtml(footnotes: Array<Footnote>): string {
  if (footnotes.length === 0) return "";

  return footnotes
    .map(
      (fn) =>
        `<div class="footnote-def" id="footnote-${fn.label}">
      <a href="#footnote-ref-${fn.label}" class="footnote-backref">${fn.label}</a>: ${marked.parseInline(fn.content)}
    </div>`,
    )
    .join("");
}
```

**결과**:

- 본문의 `[^1]` → `<sup><a href="#footnote-1">1</a></sup>`
- 페이지 하단에 각주 목록 표시
- 양방향 링크로 이동 가능

### 코드 하이라이팅 상세

`renderer.ts`에서 코드 블록에 간단한 구문 하이라이팅을 적용한다.

#### JavaScript/TypeScript 하이라이팅

```typescript
function highlightJavaScript(code: string): string {
  let highlighted = escapeHtml(formatCode(code));
  const placeholders = new Map<string, string>();
  let placeholderIndex = 0;

  // 1. 문자열 먼저 처리 (키워드 오탐 방지)
  highlighted = highlighted.replace(/(["'`])([^"'`]*?)\1/g, (match) => {
    const placeholder = `__STRING_${placeholderIndex++}__`;
    placeholders.set(placeholder, `<span class="token string">${match}</span>`);
    return placeholder;
  });

  // 2. 주석 처리
  highlighted = highlighted.replace(/\/\/.*$/gm, (match) => {
    const placeholder = `__COMMENT_${placeholderIndex++}__`;
    placeholders.set(
      placeholder,
      `<span class="token comment">${match}</span>`,
    );
    return placeholder;
  });

  // 3. 키워드 처리
  highlighted = highlighted.replace(
    /\b(const|let|var|function|import|export|from|default|createRouter|defineConfig)\b/g,
    '<span class="token keyword">$1</span>',
  );

  // 4. 숫자 처리
  highlighted = highlighted.replace(
    /\b(\d+)\b/g,
    '<span class="token number">$1</span>',
  );

  // 5. placeholder 복원
  placeholders.forEach((replacement, placeholder) => {
    highlighted = highlighted.replace(placeholder, replacement);
  });

  return highlighted;
}
```

**placeholder 패턴의 이유**: 문자열 안의 키워드(`"const"`)가 하이라이팅되지 않도록 문자열을 먼저 치환 후 복원한다.

#### 코드 렌더러

```typescript
code({ text: code, lang }: { text: string; lang?: string }) {
  let highlighted: string;

  const jsLangs = ['ts', 'js', 'javascript', 'typescript', 'jsx', 'tsx'];

  if (isJavaScriptLike(code) || (lang && jsLangs.includes(lang))) {
    highlighted = highlightJavaScript(code);
  } else if (isJSONLike(code) || lang === 'json') {
    highlighted = highlightJSON(code);
  } else {
    highlighted = escapeHtml(formatCode(code));
  }

  return `<pre><code class="${lang ? `language-${lang}` : ''}">${highlighted}</code></pre>`;
}
```

### 링크 정책

외부 링크와 내부 링크를 다르게 처리한다.

```typescript
link({ href, title, text }: { href: string; title?: string | null; text: string }): string {
  const cleanTitle = title ? ` title="${title}"` : '';
  if (!href) return `<a${cleanTitle}>${text}</a>`;

  const isExternal = /^https?:\/\//i.test(href);
  const target = isExternal ? ' target="_blank" rel="noopener noreferrer"' : '';

  return `<a href="${href}"${cleanTitle}${target}>${text}</a>`;
}
```

- **외부 링크** (`https://...`): 새 탭에서 열림 + `noopener noreferrer`로 보안 강화
- **내부 링크**: 같은 탭에서 열림

### 보안: DOMPurify 상세

프로덕션 환경에서는 XSS 공격 방지를 위해 DOMPurify로 HTML을 정화한다.

```typescript
export const SANITIZE_OPTIONS = {
  ADD_ATTR: ["target", "rel", "open", "class", "href", "id"],
  ADD_TAGS: ["span", "sup", "a"],
  ALLOWED_URI_REGEXP:
    /^(?:(?:https?|mailto|tel|data:image\/(?:png|gif|jpeg|webp));|\/|#)/i,
};
```

| 옵션                 | 설명                               |
| -------------------- | ---------------------------------- |
| `ADD_ATTR`           | 허용할 HTML 속성                   |
| `ADD_TAGS`           | 허용할 HTML 태그                   |
| `ALLOWED_URI_REGEXP` | 허용할 URI 패턴 (javascript: 차단) |

**개발 환경에서 비활성화하는 이유**:

- 빠른 렌더링 (DOMPurify 오버헤드 제거)
- 개발 중 XSS 테스트 가능

### 표(Table) 렌더링

GFM(GitHub Flavored Markdown) 표 문법을 지원한다. marked의 `gfm: true` 옵션으로 기본 지원되며, CSS로 스타일링한다.

#### 문법

```markdown
| 필드  | 설명      |
| ----- | --------- |
| title | 노트 제목 |
| tags  | 태그 배열 |
```

#### 정렬 지원

```markdown
| 왼쪽 | 가운데 | 오른쪽 |
| :--- | :----: | -----: |
| L    |   C    |      R |
```

- `:---` : 왼쪽 정렬
- `:---:` : 가운데 정렬
- `---:` : 오른쪽 정렬

#### 스타일링 (CSS)

```css
.markdown-content table {
  width: 100%;
  border-collapse: collapse;
  font-size: 14px;
  margin: 16px 0;
  border: 1px solid var(--border);
  border-radius: 8px;
  overflow: hidden;
}

.markdown-content table thead {
  background-color: var(--muted);
}

.markdown-content table th {
  padding: 10px 14px;
  font-weight: 600;
  border-bottom: 2px solid var(--border);
}

.markdown-content table td {
  padding: 10px 14px;
  border-bottom: 1px solid var(--border);
}

.markdown-content table tbody tr:hover {
  background-color: color-mix(in oklab, var(--muted) 50%, transparent);
}
```

**설계 결정**: marked의 table 렌더러 API가 토큰 객체를 전달하므로, 커스텀 렌더러 대신 기본 렌더러를 사용하고 CSS로 스타일링한다.

### 구분선(Horizontal Rule) 렌더링

마크다운 구분선을 커스텀 스타일로 렌더링한다.

#### 문법

```markdown
---
```

또는 `***`, `___`도 가능하다.

#### 커스텀 렌더러

```typescript
hr(): string {
  return '<hr class="markdown-hr" />'
}
```

#### 스타일링 (CSS)

```css
.markdown-content .markdown-hr {
  border: 0.5px solid var(--border);
  margin: 72px 0;
}
```

넓은 여백(72px)으로 섹션 구분을 명확하게 한다.
