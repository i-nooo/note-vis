- 간단한 패턴 매칭과 추출에 집중하고, 복잡한 로직은 일반 코드로 처리합니다.

### 예시

- 이메일
  - `local parts` + @ + `domain` [^1]
  - ID + at + 해당 ID를 발급한 메일 서버

  - local parts:
    - 3자 이상
    - 영어만
    - 대소문자 구분 있음

  - domain parts:
    - 3자 이상
    - \*.\* 형식
    - 대소문자 구분 없음

```javascript
^[A-Za-z]{3,}@(?i:(?=.{3,}$)[a-z]+(?:\.[a-z]+)+)$
```

- ^[A-Za-z]{3,}: local part는 영어만, 3자 이상(대소문자 구분은 정규식 기본 동작으로 유지)
- @: 구분자
- (?i: ... ): 이 괄호 안(도메인)만 대소문자 구분 없이 매칭
- (?=.{3,}$): 도메인 전체 길이 3자 이상
- [a-z]+(?:\.[a-z]+)+: _._ 형태 보장(점이 최소 1개 이상, 즉 a.b / a.b.c 등 가능)

[^1]: https://datatracker.ietf.org/doc/html/rfc5321
