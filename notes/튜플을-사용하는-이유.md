튜플은 배열(list)과 딕셔너리(dict)와는 다른 역할과 성질을 갖는다.

<br/>

1. 불변성
2. 이질적 데이터
3. 성능 및 최적화
4. 함수 반환값 패턴

<br/>

## 1. 불변성

- 정의된 값은 수정이 불가능하다는 불변성 때문에 해시 가능(hashable)한 객체로 쓸 수 있다.
- 덕분에 딕셔너리의 키나 집합의 원소로 사용할 수 있다.
- 리스트는 불변적이지 않기 때문에 이 역할을 수행할 수 없다.

<br/>

## 2. 이질적 데이터 [^1]

- 리스트는 동질적 데이터 집합을 다루는데 주로 쓰인다.
  - 학생 점수 목록, 상품 ID 등
- 반면 튜플은 이질적 데이터 구조를 표현하는데 적합하다.
  - (이름, 나이, 직업)처럼 여러 속성이 묶인 하나의 레코드
- 즉, 순서가 있는 불변 구조체로, 작은 데이터 구조체의 역할을 대신할 수 있다.

<br/>

## 3. 성능 및 최적화

- 불변이라는 성질 때문에 리스트보다 메모리를 적게 쓰고, 접근 속도도 조금 더 빠르다.

<br/>

## 4. 함수 반환값 패턴

- 함수에서 여러 값을 동시에 반환할 때 쓰이는 구조가 튜플이다.

```python
def divide(a, b):
    return a//b, a%b

# 반환값: (a//b, a%b)
```

- 호출 시 자동으로 언패킹이 가능하여 (몫, 나머지)처럼 두 값이 명확하게 묶인다.

[^1]: 동질적, 이질적: 데이터나 집단의 구성요소가 서로 같거나 다른 정도를 설명하는 개념. 이질적은 "성질이 다른 것들로 이루어져 있다."를 의미한다.
